fluid:
language-integrated data provenance

Fluid is a new programming language which utilises program analysis/data provenance techniques to connect outputs to data sources in a fine-grained way. For example, selecting a particular bar in a bar chart generated from an underlying table will automatically present a view of the table containing only the data needed to compute that bar. The resulting data selection is a restricted view of a dataset where only relevant rows are shown, and the relevant cells within those rows are highlighted.

data-linked visualisations

Try this out by selecting the bar for Germany in the left-hand chart, or one of the data points for year 2018 in the right-hand chart:

Hand-crafting a view like this would involve manually tracking the dependencies between input and output, something which Fluid does automatically. The programmer simply expresses the bar chart and line chart as pure functions of some common data:

bar-chart.fld and line-chart.fld

As well as offering the first fully automated (and thus potentially pervasive) solution to this well-recognised problem in data visualisation, our approach to data linking is “transparent”, in that a view of the specific data elements which “explain” why the two visual elements are linked can always be made available.
